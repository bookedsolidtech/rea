# Migrating to rea 0.26.0

> Audience: anyone running rea 0.25.x or earlier as a consumer.
> Internal repos (BST, helixir) follow this migration too.

## TL;DR

0.26.0 makes local-first delegation **forceful** — the agent literally
cannot push until codex has reviewed the working tree. There is a
clean off-switch for teams without codex/claude installed.

## What's new

Three enforcement layers, all calling the new `rea preflight`:

1. `local-review-gate.sh` — Bash-tier PreToolUse hook that refuses
   `git push` (and optionally `git commit`) from Claude Code's Bash
   tool.
2. `.husky/pre-push` — now runs `rea preflight --strict` BEFORE the
   push-gate dispatch. Refuses `git push` at the terminal layer.
3. New CLIs: `rea review` (writes audit entry), `rea preflight`
   (reads audit entry).

## Off-switch — for teams without codex/claude

This is the FIRST-class concern. If your team doesn't have codex
installed (or doesn't run codex/claude in their workflow), set the
off-switch in `.rea/policy.yaml` BEFORE upgrading:

```yaml
review:
  local_review:
    mode: off
```

When `mode: off`:

- The Bash-tier `local-review-gate.sh` hook is a silent no-op.
- The husky pre-push `rea preflight --strict` exits 0 immediately.
- `rea review` exits 0 silently when codex is also missing
  (audited as `rea.local_review.skipped_unavailable`).
- The push-gate (`policy.review.codex_required`) is unaffected —
  it has its own off-switch.

If you upgrade WITHOUT setting `mode: off` and don't have codex,
`git push` will refuse with a helpful message naming the off-switch.
Nothing is broken; the gate is just doing its job.

## Default behavior (codex available)

`mode: enforced` is the default when unset. Every `git push` (and
optionally `git commit`) requires a recent `rea.local_review` audit
entry covering HEAD.

The flow:

```bash
# 1. edit working tree
vim src/foo.ts

# 2. run local review
rea review
# → writes rea.local_review audit entry with current HEAD sha

# 3. commit + push
git commit -s -m "fix: ..."
git push
# → husky pre-push runs `rea preflight --strict`
# → finds the matching audit entry, exits 0
# → push-gate runs as the second layer (codex on push)
```

If you push WITHOUT running `rea review` first:

```bash
git push
# rea: preflight refuse — no recent local-review audit entry covers HEAD
#
#   To unblock, do ONE of:
#     1. Run `rea review`           — write a fresh local-review audit entry
#     2. Set REA_SKIP_LOCAL_REVIEW="<reason>"
#                                   — per-invocation override (audited)
#     3. Edit .rea/policy.yaml      — set:
#          review:
#            local_review:
#              mode: off
```

## Per-invocation override

For the rare urgent push that can't wait for review:

```bash
REA_SKIP_LOCAL_REVIEW="prod hotfix #1234" git push
# → preflight allows
# → audit log records `rea.local_review.skipped_override`
#   with `reason: "prod hotfix #1234"`
```

This is a release valve, not a sustained way to disable enforcement.
Every override is audited.

## Commit-hygiene refusal

New: `rea preflight` also refuses when `git rev-list --count
<base>..HEAD > policy.commit_hygiene.refuse_at_commits` (default 5).
Warns at `> warn_at_commits` (default 1). Husky pre-push runs
`--strict` so warns become refusals.

To raise the thresholds:

```yaml
commit_hygiene:
  warn_at_commits: 5
  refuse_at_commits: 20
```

## Audit-data migration

Pre-0.26.0 audit entries use `tool_name: codex.review`. `rea preflight`
accepts these AS-IS — no audit-log rewrite needed. Existing reviews
keep their coverage.

## Upgrade steps

```bash
# 1. (only if your team doesn't have codex) set the off-switch
vim .rea/policy.yaml      # add review.local_review.mode: off

# 2. upgrade rea
pnpm up @bookedsolid/rea

# 3. refresh husky pre-push + .claude/settings.json
rea upgrade

# 4. verify
rea doctor
rea preflight             # exit 0 if all clear
```

## Verifying the install

```bash
# the new hook is present:
ls .claude/hooks/local-review-gate.sh

# the new CLIs work:
rea review --help
rea preflight --help

# the husky pre-push body has v5 markers:
head -3 .husky/pre-push
# → #!/bin/sh
# → # rea:husky-pre-push-gate v5
# → # rea:gate-body-v5
```

## Rolling back

If a 0.26.0 upgrade breaks your workflow and you need to bypass
quickly while you investigate:

```bash
# per-shell bypass (audited):
export REA_SKIP_LOCAL_REVIEW="rollback investigation"

# OR temporarily disable for the team:
# .rea/policy.yaml:
#   review:
#     local_review:
#       mode: off
```

Both are reversible. There is no audit-data destruction in 0.26.0.
