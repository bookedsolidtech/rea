Consumer-UX bundle: `rea upgrade --check` + `rea audit summary` + README refresh.

After the hook-port marathon (0.32→0.35) and five hardening/polish
releases (0.36→0.40), the hook system is structurally complete.
0.41.0 turns the focus to consumer-facing surfaces — three
user-visible items that make the platform's existing capabilities
easier to reach.

## 1. `rea upgrade --check`

New non-interactive preview mode for `rea upgrade`:

```bash
rea upgrade --check              # text summary + unified diffs, exit 0
rea upgrade --check --json       # machine-readable preview document
rea upgrade --check --no-diff    # paths + counts only (large repos)
```

`--check` classifies every file `rea upgrade` would touch and emits
a summary table (created / modified / unchanged / removed-upstream)
plus a unified diff per modified file. Always exits 0 — it's a
preview, not an enforcement gate. Wire it into CI to surface the
changes a `rea upgrade` PR would produce before merging.

Distinct from `--dry-run`: `--dry-run` rehearses the FULL
interactive flow with writes suppressed (prompts still fire, output
streams in classification order). `--check` is purely structured
and non-interactive, and emits the unified diffs that `--dry-run`
does not. Mirrors `terraform plan` / `npm install --dry-run`.

New surface:

- `src/cli/upgrade-check.ts` — `computeUpgradeCheck()`,
  `renderUpgradeCheck()`, `runUpgradeCheck()` (the test seam +
  renderer + commander entrypoint, respectively).
- `src/cli/install/unified-diff.ts` — minimal LCS-based unified-diff
  helper (no new npm dep). Files are bounded at 256 KiB by callers
  AND by `MAX_LCS_CELLS` line-count guard (see codex round-1 fix
  below).
- `--check` / `--json` / `--no-diff` flags wired into the existing
  `rea upgrade` command in `src/cli/index.ts`. The real upgrade
  flow is unchanged when `--check` is absent.

## 2. `rea audit summary [--since=DUR] [--json]`

New high-level audit-log overview reader:

```bash
rea audit summary                     # all-time rollup
rea audit summary --since 24h         # last 24h (units: s/m/h/d/w)
rea audit summary --since 7d --json   # jq-friendly document
```

Counts events by `tool_name` / tier / status / session, reports the
time window covered, and runs a sample-verified chain-integrity
check (up to 12 samples per window). `--since` accepts a compact
duration (`24h`, `7d`, `90m`, `2w`) — distinct from `--since
<rotated-file>` on `rea audit verify` / `rea audit specialists`,
which anchors on a rotated-audit basename. Two `--since` semantics
serve different needs (anchor for chain walks, duration for
summarization); we accept the surface area cost.

Text output is the default; `--json` emits a stable document
(`schema_version: 1`). Tier-Read; never mutates the chain.

New surface:

- `src/cli/audit-summary.ts` — `computeAuditSummary()`,
  `parseDurationSeconds()`, `renderAuditSummary()`,
  `runAuditSummary()`, `registerAuditSummaryCommand()`.
- Subcommand registered under the existing `rea audit` group in
  `src/cli/index.ts`, next to `rotate` / `verify` / `specialists`.

## Codex round-1 fixes folded in (1 P1, 2 P2)

Pre-commit codex review surfaced three correctness gaps; all three
landed in the same squashed commit:

- **P1 — unified-diff line-count guard.** The 256 KiB byte cap
  callers enforce did not protect against pathological line counts:
  a ~200 KiB file of one-character lines (200K lines) would have
  tried to allocate ~160 GiB of `Uint32Array` cells for the LCS
  table and hung the process. Added a `MAX_LCS_CELLS` cap
  (~16 MiB at 4 bytes/cell). Inputs over the cap return a
  `DIFF_TOO_LARGE_NOTICE` sentinel which `safeDiff()` in
  `upgrade-check.ts` detects and surfaces as `diff_truncated: true`.

- **P2 — `rea audit summary` walks rotated files by default.** Pre-
  fix, omitting `--since` walked the current `.rea/audit.jsonl`
  only while the header still rendered "all time". Long-lived
  repos with rotated history were silently undercounted. Post-fix
  the no-`--since` path includes every rotated segment plus the
  current file. The `--since` path is unchanged (it already
  honored rotated history within the window).

- **P2 — `.gitignore` synthetic row in `rea upgrade --check`.**
  `runUpgrade()` calls `ensureReaGitignore()` to backfill the
  managed block for older installs missing newer entries
  (`.rea/last-review.json`, `.rea/fingerprints.json`, etc.). The
  preview omitted this entry, so the planner could report a
  fully-in-sync repo and then `rea upgrade` would still mutate
  `.gitignore` — breaking the preview contract. Added a
  `classifyGitignore()` synthetic entry. `ensureReaGitignore()`
  gained a `dryRun: true` option that returns `previewContent` +
  `previousContent` instead of writing, preserving the pre-0.41.0
  call shape via a back-compat positional-args branch.
