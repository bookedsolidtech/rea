feat(0.39.0): policy-reader doctor checks — surface 4-tier ladder reachability

Adds five new `rea doctor` checks that report which tiers of the
0.37.0 `hooks/_lib/policy-reader.sh` ladder are reachable in the
operator's environment. Pre-0.39.0 operators had no way to see whether
their shims would silently no-op flow-form policy
(`local_review: { mode: off }`, `blocked_paths: [.env, ...]`) when the
rea CLI is unreachable AND python3+PyYAML are missing — Tier 3 (awk)
only handles block-form.

New per-tier probes (`checkPolicyReaderTier1` / `Tier2` / `Tier3` /
`Jq` / `TierSummary` in `src/cli/doctor.ts`), all exported and
accepting an optional `PolicyReaderProbes` injection for deterministic
unit tests:

- Tier 1: `rea hook policy-get` — verifies the CLI dist is present at
  one of the two shim-resolved paths (`node_modules/@bookedsolid/rea/
  dist/cli/index.js` or `<baseDir>/dist/cli/index.js`) AND actually
  responds to `rea hook policy-get version --json` (the same probe
  `_pr_load_full_json` in policy-reader.sh uses). `pass` when dist is
  present + invokable; `warn` (stale/broken) when dist exists but the
  CLI fails to respond; `warn` (not installed) when dist is absent.
  Round-1 P2 (2026-05-16): the file-existence-only probe alone would
  falsely report `pass` for a stale dist while the real ladder skipped
  Tier 1.
- Tier 2: `python3` + stdlib `yaml` (PyYAML). `pass` when both
  reachable; `warn` when either is missing (the loader falls through
  to Tier 3 awk, block-form only — flow-form silently no-ops when CLI
  is also unreachable).
- Tier 3: `awk`. Almost always passes (POSIX requirement). Hard `fail`
  when truly absent — zero working fallback tiers means every shim
  policy lookup fails closed.
- `jq`: optional JSON accelerator. `pass` when on PATH; `warn` when
  absent (per the 0.37.0 round-1 P2 fix the helper falls back to a
  python3 walker — correct, just an extra spawn per leaf).
- Effective floor: aggregate roll-up — `pass` when Tier 1 OR Tier 2
  reachable AND a JSON list walker (jq or python3) is available;
  `warn` (flow-form-lists-degraded) when Tier 1 reachable but neither
  jq nor python3 on PATH (flow-form scalars work via the CLI's JSON
  output, but `policy_reader_get_list` cannot iterate the resulting
  JSON arrays — flow-form `blocked_paths: [.env, ...]` silently
  falls through to Tier 3 awk); `warn` (Tier-3-only) when only awk
  reachable (silent no-op risk on both scalars and lists); `fail`
  when no tiers reachable. Round-1 P2 (2026-05-16) added the
  list-walker dimension — pre-fix the summary reported `pass` for
  the "Tier 1 + no jq + no python3" shape while flow-form lists
  silently no-op'd.

Probes scrub the same `PYTHONPATH` / `PYTHONHOME` / `PYTHONSTARTUP`
env vars + set `PYTHONSAFEPATH=1` that policy-reader.sh applies, so
the probe verdict can't be falsely positive against a repo where the
real loader would (correctly) refuse to import.

Gated on `.rea/policy.yaml` existence — when the policy file is
absent `checkPolicyParses` already fails and the tier checks would
add no signal. 30 new unit tests in `src/cli/doctor.test.ts` (80
total — up from 50) covering every probe combination via injected
stubs (no PATH manipulation, no real subprocess spawns).

No changes to `hooks/_lib/policy-reader.sh` itself — observational
visibility only.
